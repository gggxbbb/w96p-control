# W96P 控制

> W96P / W66D 蓝牙风扇 Web 上位机 · 基于 Web Bluetooth API · 浏览器直连设备

## 协议文档

协议基于 Bluetooth LE GATT，所有多字节整数均使用**大端序（Big-Endian）**。

### 1. 概述

| 项目 | 值 |
|------|-----|
| 设备广播名 | `W96P` |
| 连接方式 | `navigator.bluetooth.requestDevice` |
| 服务数量 | 3 个主服务 |
| 数据编码 | 主控·自然风：二进制 hex；电源：ASCII 文本命令（以 `,` 结尾） |
| 字节序 | 大端（Big-Endian），`DataView.getUint16(off, false)` |

### 1.1 服务列表

| 服务名称 | UUID | 用途 |
|----------|------|------|
| 主控服务 Main Service | `0000fff0-0000-1000-8000-00805f9b34fb` | 风扇开关、档位、转速、定时、自然风、休眠 |
| 电源服务 FFD0 Service | `0000ffd0-0000-1000-8000-00805f9b34fb` | 电池、电源状态、电机、快充配置 |
| 自然风服务 Nature Wind Service | `0000ffe0-0000-1000-8000-00805f9b34fb` | 自然风曲线数据 |

### 1.2 连接流程

1. 以 `filters: [{ name: "W96P" }]` 扫描设备，`optionalServices` 声明三个服务 UUID。
2. 连接 GATT 后依次获取三个 PrimaryService。
3. 缓存所有 Characteristic，连接完成后延迟约 1.5s 读取初始状态（剩余时间、档位风速、自然风状态、休眠延时、减档模式），再启动 1 秒周期的实时刷新。
4. 断开时监听 `gattserverdisconnected` 事件，清理缓存与定时器。

### 2. 主控服务特征（Main Service `FFF0`）

本服务下所有特征采用**二进制 hex 字节**写入（`writeValue`）。

#### 2.1 POWER — 开关机/档位 `FFF1`

| 项目 | 内容 |
|------|------|
| UUID | `0000fff1-0000-1000-8000-00805f9b34fb` |
| 属性 | Write |
| 数据长度 | 1 字节 |

**取值定义：**

| 值（hex） | 含义 |
|-----------|------|
| `00` | 关机 |
| `01` | 1 档 |
| `02` | 2 档 |
| `03` | 3 档 |
| `04` | 4 档 |

**注意：** 设置档位时若自然风处于开启状态，需先写 `NATURE_WIND = 00` 关闭自然风，间隔约 100ms 后再写档位命令。

**示例：** 开启 2 档 → 写入字节 `0x02`

#### 2.2 TIMER — 定时关机 `FFF2`

| 项目 | 内容 |
|------|------|
| UUID | `0000fff2-0000-1000-8000-00805f9b34fb` |
| 属性 | Read / Write |
| 数据长度 | 2 字节 |

**数据格式：** `uint16` 大端，单位**秒**。

| 操作 | 写入值 |
|------|--------|
| 取消定时 | `00 00` |
| 设置 N 分钟 | 将 `N × 60` 秒转为 4 位 hex（大端） |

**取值范围：** 1~480 分钟（应用层限制），`0` = 取消定时。

**读取：** 返回剩余秒数，按 `分X秒` 显示。

**示例：** 设置 30 分钟 → 1800 秒 = `0x0708` → 写入字节 `07 08`

#### 2.3 FAN_SPEED — 风扇转速 `FFF3`

| 项目 | 内容 |
|------|------|
| UUID | `0000fff3-0000-1000-8000-00805f9b34fb` |
| 属性 | Read / Write（优先 `writeWithoutResponse`） |
| 数据长度 | 1 字节 |

**数据格式：** 单字节无符号整数，表示转速百分比。

| 范围 | 含义 |
|------|------|
| 0~100 | 转速百分比 |

**写入策略：**
- 若特征支持 `writeWithoutResponse`，优先使用（拖动滑块时实时性更好）。
- 设置转速时若自然风开启，需先关闭自然风（写 `NATURE_WIND = 00`）。

**读取：** 返回当前转速百分比（0~100）。

**示例：** 设置 75% 转速 → 写入字节 `0x4B`

#### 2.4 NATURE_WIND — 自然风开关 `FFF4`

| 项目 | 内容 |
|------|------|
| UUID | `0000fff4-0000-1000-8000-00805f9b34fb` |
| 属性 | Read / Write |
| 数据长度 | 1 字节 |

**取值定义：**

| 值（hex） | 含义 |
|-----------|------|
| `00` | 关闭自然风 |
| `01` | 开启自然风 |

**读取：** `1` = 开启，其他 = 关闭。

**互斥关系：** 开启档位（POWER）或调整转速（FAN_SPEED）时会自动关闭自然风。

#### 2.5 SHUTDOWN_DELAY — 蓝牙自动休眠延时 `FFF5`

| 项目 | 内容 |
|------|------|
| UUID | `0000fff5-0000-1000-8000-00805f9b34fb` |
| 属性 | Read / Write |
| 数据长度 | 2 字节 |

**数据格式：** `uint16` 大端，单位**秒**。

| 写入值 | 含义 |
|--------|------|
| `00 00` | 永不自动休眠 |
| 1~65535 | N 秒后自动关闭蓝牙 |

**应用层规则：**
- 输入 1~9 会被自动修正为 10。
- 有效范围 1~65535 秒。

**读取：** 返回当前休眠延时秒数；`0` = 永不关闭。

**示例：** 设置 60 秒后休眠 → 写入字节 `00 3C`

#### 2.6 GEAR_DOWN_MODE — 减档模式 `FFF6`

| 项目 | 内容 |
|------|------|
| UUID | `0000fff6-0000-1000-8000-00805f9b34fb` |
| 属性 | Read / Write |
| 数据长度 | 1 字节 |

**取值定义：**

| 值（hex） | 含义 |
|-----------|------|
| `00` | 逐级减档（按档位风速校准值逐级降低） |
| `01` | 直接回 0（直接关停） |

**读取：** 返回当前模式（0 或 1）。

#### 2.7 SPEED_CALIB — 档位风速校准 `FFF7`

| 项目 | 内容 |
|------|------|
| UUID | `0000fff7-0000-1000-8000-00805f9b34fb` |
| 属性 | Read / Write |
| 数据长度 | 4 字节 |

**数据格式：** 4 个字节，依次对应 1~4 档的风速百分比。

| 字节偏移 | 含义 | 范围 |
|----------|------|------|
| 0 | 1 档风速 | 0~100（应用层建议 ≥20） |
| 1 | 2 档风速 | 0~100 |
| 2 | 3 档风速 | 0~100 |
| 3 | 4 档风速 | 0~100 |

**默认值：** `1E 32 46 64`（30% / 50% / 70% / 100%）

**示例：** 设置为 25%/45%/65%/90% → 写入 `19 2D 41 5A`

### 3. 自然风服务特征（Nature Wind Service `FFE0`）

#### 3.1 NATURE_WIND_CURVE — 自然风曲线 `FFE3`

| 项目 | 内容 |
|------|------|
| UUID | `0000ffe3-0000-1000-8000-00805f9b34fb` |
| 属性 | Read / Write（优先 `writeWithoutResponse`） |
| 数据长度 | 128 字节 |

**数据格式：** 128 个数据点，每个字节表示一个风速强度值。

| 字节偏移 | 含义 | 范围 |
|----------|------|------|
| 0~127 | 128 个风速强度采样点 | 0~100 |

**写入：** 将 128 个十进制值各转为 1 字节 hex，拼接后整体写入（共 128 字节）。

**读取：** 读取 128 字节，每字节还原为风速强度。

**说明：**
- 即使自然风关闭，也可修改和保存曲线，开启自然风后生效。

### 4. 电源服务特征（FFD0 Service）

本服务下特征用于读取电源/电池/电机状态，以及通过 **ASCII 文本命令**配置快充参数。所有 ASCII 命令均以 `,`（逗号）结尾，用 `TextEncoder` 编码为 UTF-8 字节后 `writeValue`。

#### 4.1 BATTERY_INFO — 电池信息 `FFD1`

| 项目 | 内容 |
|------|------|
| UUID | `0000ffd1-0000-1000-8000-00805f9b34fb` |
| 属性 | Read / Write（ASCII 命令） |
| 读取长度 | 8 字节 |

**读取数据格式：**

| 字节偏移 | 类型 | 含义 | 单位 |
|----------|------|------|------|
| 0~1 | `uint16` BE | 电池电压 | mV |
| 2~3 | `int16` BE | 电池电流（有符号，正=充电，负=放电） | mA |
| 4~7 | `uint32` BE | 电池容量 | mWh |

**写入命令（设置容量）：**

| 命令 | 说明 |
|------|------|
| `BAT_CAP=<mWh>,` | 设置电池容量，`<mWh>` = 额定容量(mAh) × 额定电压(V) |

**应用层限制：**
- 容量范围：100~50000 mAh
- 电压范围：3~30 V
- 计算后 mWh 范围：1000~50000

**示例：** 5000mAh × 3.6V = 18000 mWh → 写入 ASCII `BAT_CAP=18000,`

#### 4.2 POWER_STATUS — 电源状态 `FFD2`

| 项目 | 内容 |
|------|------|
| UUID | `0000ffd2-0000-1000-8000-00805f9b34fb` |
| 属性 | Read / Write（ASCII 命令） |
| 读取长度 | ≥11 字节 |

**读取数据格式：**

| 字节偏移 | 类型 | 含义 | 说明 |
|----------|------|------|------|
| 0~3 | `uint32` BE | VBUS 电压 | mV |
| 4~5 | `int16` BE | VBUS 电流 | mA（有符号） |
| 6 | `uint8` | POW_C | 电路通断状态（1=开启，0=关闭） |
| 7 | `uint8` | POW_STA | 充放电状态（1=充电中，其他=放电中） |
| 8 | `uint8` | POW_C_OUT | C 口输出快充使能（0=使能，1=关闭） |
| 9 | `uint8` | POW_C_IN | C 口输入快充使能（0=使能，1=关闭） |
| 10 | `uint8` | POW_C_HI | 保留（读取但不使用） |

**写入命令：**

| 命令 | 说明 |
|------|------|
| `POW_C_OUT=<0\|1>,` | C 口输出快充（0=使能，1=关闭） |
| `POW_C_IN=<0\|1>,` | C 口输入快充（0=使能，1=关闭） |

**示例：** 关闭 C 口输出快充 → 写入 ASCII `POW_C_OUT=1,`

#### 4.3 MOTOR_INFO — 电机信息 `FFD3`

| 项目 | 内容 |
|------|------|
| UUID | `0000ffd3-0000-1000-8000-00805f9b34fb` |
| 属性 | Read |
| 读取长度 | ≥4 字节（电压字段位置因版本略有差异） |

**读取数据格式（V3.1）：**

| 字节偏移 | 类型 | 含义 | 说明 |
|----------|------|------|------|
| 0~1 | `uint16` BE | 电机电流 | mA |
| 2 | `uint8` | 堵转状态 | 掩码 `0xF7`（清除 bit3 保留位），值为 1 表示堵转 |
| 4~5 | `uint16` BE | 电机电压 | mV |

**读取数据格式（V3.2）：**

| 字节偏移 | 类型 | 含义 | 说明 |
|----------|------|------|------|
| 0~1 | `uint16` BE | 电机电流 | mA |
| 2 | `uint8` | 电机堵转/block | 原始字节 |
| 末尾 2 字节 | `uint16` BE | 电机电压 | mV（取 `byteLength - 2` 偏移） |

> 电压范围校验：0~20000 mV，超出视为脏数据置 0。

**派生计算：**
- 电机功率 (W) = 电机电压 (V) × 电机电流 (A)
- 电池功率 (W) = 电池电压 (V) × 电池电流 (A)，正值=充电，负值=放电

#### 4.4 POWER_CONFIG — 快充配置 `FFD4`

| 项目 | 内容 |
|------|------|
| UUID | `0000ffd4-0000-1000-8000-00805f9b34fb` |
| 属性 | Read / Write（ASCII 命令） |
| 读取长度 | ≥16 字节 |

**读取数据格式：**

| 字节偏移 | 字段 | 含义 |
|----------|------|------|
| 1 | POW_VER | PD 版本（bit6：0=PD3.0，1=PD2.0） |
| 2 | POW_SINK | 当前输入快充协议 |
| 3 | POW_SRC | 当前输出快充协议 |
| 6 | POW_1A | 输入/输出电压与协议支持（位域） |
| 7 | POW_1C | PD/FCP Source/Sink 使能（位域） |
| 8 | POW_1D | AFC/FCP/SCP/SFCP Source/Sink 使能（位域） |
| 9 | POW_1E | QC Source 使能（位域） |
| 13 | POW_2A | PD 版本与 PPS1 电压（位域） |
| 14 | POW_2B | PD Fix 电压、PPS0/PPS1、重新广播（位域） |
| 15 | POW_2C | 5V/9V/12V PDO 电流（位域） |

**POW_SINK（输入快充协议）枚举：**

| 值 | 协议 |
|----|------|
| 0 | 非快充 |
| 1 | PD Sink |
| 3 | HV Sink |
| 4 | AFC Sink |
| 5 | FCP Sink |
| 6 | SCP Sink |
| 7 | PE1.1 Sink |

**POW_SRC（输出快充协议）枚举：**

| 值 | 协议 |
|----|------|
| 0 | 非快充 |
| 1 | PD Source |
| 2 | PPS Source |
| 3 | QC2.0 Source |
| 4 | QC3.0 Source |
| 5 | FCP Source |
| 6 | PE2.0/1.1 Source |
| 7 | SFCP Source |
| 8 | AFC Source |
| 9 | SCP Source |
| 10 | LVDC1 Source |

##### POW_1A 位定义

| Bit | 含义 | 0 | 1 |
|-----|------|---|---|
| bit1 | 输出最高电压 | 12V | 9V（FCP/PD 无效） |
| bit2 | 输入支持 12V | 支持 | 不支持 |
| bit3 | FCP 输出 12V 支持 | 不支持 | 支持 |
| bit4 | AFC 输出 12V 支持 | 不支持 | 支持 |

默认值：`0x1C`（`00011100`，AFC/FCP 输出 12V 支持，输入不支持 12V）

##### POW_1C 位定义

| Bit | 含义 | 0 | 1 |
|-----|------|---|---|
| bit0 | FCP Source 使能 | 使能 | 不使能 |
| bit4 | PD Sink 使能 | 使能 | 不使能 |
| bit5 | PD Source 使能 | 使能 | 不使能 |

默认值：`0x00`（全部使能）

##### POW_1D 位定义

| Bit | 含义 | 0 | 1 |
|-----|------|---|---|
| bit0 | SFCP Source 使能 | 使能 | 不使能 |
| bit1 | SCP Sink 使能 | 使能 | 不使能 |
| bit2 | SCP Source 使能 | 使能 | 不使能 |
| bit3 | AFC Sink 使能 | 使能 | 不使能 |
| bit4 | AFC Source 使能 | 使能 | 不使能 |
| bit7 | FCP Sink 使能 | 使能 | 不使能 |

默认值：`0x00`（全部使能）

##### POW_1E 位定义

| Bit | 含义 | 0 | 1 |
|-----|------|---|---|
| bit1 | QC Source 使能 | 使能 | 不使能 |

默认值：`0x00`

##### POW_2A 位定义

| Bit | 含义 | 0 | 1 |
|-----|------|---|---|
| bit5 | PPS1 最高电压 | 11V | 9V |
| bit6 | PD 版本 | PD3.0 | PD2.0 |

默认值：`0x00`（PD3.0，PPS1 11V）

##### POW_2B 位定义

| Bit | 含义 | 0 | 1 |
|-----|------|---|---|
| bit3 | PPS1 使能 | 使能 | 不使能 |
| bit4 | PD 重新广播 5V/2A | 不使能 | 使能 |
| bit5 | PPS0 使能 | 使能 | 不使能 |
| bit7 | PD Fix 输出电压 | 12V | 9V |

默认值：`0x10`（重新广播使能）

##### POW_2C 位定义（PDO 电流）

| Bit | 含义 | 0 | 1 | 2 | 3 |
|-----|------|---|---|---|---|
| bit0~1 | 12V PDO 电流 | 1.5A | 1.6A | 1.7A | 1.75A |
| bit2~3 | 9V PDO 电流 | 2.0A | 2.2A | 2.33A | 2.4A |
| bit4~5 | 5V PDO 电流 | 3.0A | 2.4A | 2.5A | 2.0A |

默认值：`0x04`（`00000100`，5V=3.0A，9V=2.2A，12V=1.5A）

**写入命令：**

| 命令 | 说明 |
|------|------|
| `POW_1A=<byte>,` | 设置 POW_1A 寄存器 |
| `POW_1C=<byte>,` | 设置 POW_1C 寄存器 |
| `POW_1D=<byte>,` | 设置 POW_1D 寄存器 |
| `POW_1E=<byte>,` | 设置 POW_1E 寄存器 |
| `POW_2A=<byte>,` | 设置 POW_2A 寄存器 |
| `POW_2B=<byte>,` | 设置 POW_2B 寄存器 |
| `POW_2C=<byte>,` | 设置 POW_2C 寄存器 |

> 写入时需先读取当前寄存器值，按位掩码修改目标位后整体写入，避免覆盖其他位。

**示例：** 将 PD 版本设为 PD2.0（POW_2A bit6=1）→ 若当前 `0x00`，新值 `0x40` → 写入 ASCII `POW_2A=64,`

### 6. 通信注意事项

1. **写入互斥**：主控服务写入需串行化，避免并发 `writeValue` 冲突；应用层用 `isWriting`/`isPowerSending` 等标志加队列。
2. **自然风互斥**：开启档位或调整转速前必须先关闭自然风，间隔约 100ms。
3. **实时刷新**：连接后每 1 秒轮询转速、电池、电源、电机、自然风、减档模式状态；拖动滑块或写入操作期间应暂停刷新，完成后延迟约 300ms 恢复。
4. **ASCII 命令格式**：电源服务所有写命令均为 `KEY=VALUE,`（末尾逗号），UTF-8 编码。
5. **位域写入**：POW_1A/1C/1D/1E/2A/2B/2C 为位域寄存器，写入前必须读-改-写，用掩码保留其他位。
6. **重试机制**：单次写入失败建议重试 1~3 次，间隔 200~500ms。
7. **电压校验**：电机电压超出 0~20000 mV 视为脏数据，置 0 显示。
8. **可见性优化**：页面隐藏（`visibilitychange`）时停止实时刷新，恢复时按连接状态重启。

## 多设备兼容

W96P 与 W66D 协议层完全兼容（命令可互发），但业务调校不同：

| 差异 | W96P | W66D |
|------|------|------|
| 默认档位 | 10/35/70/100 | 30/50/70/100 |
| 风速范围 | 0-100 | 20-90 |
| 电机解析 | 电流+堵转+电压 | 仅电流 |

`profiles.ts` 按 `device.name` 加载对应调校包，未知型号走 W66D 保守包。

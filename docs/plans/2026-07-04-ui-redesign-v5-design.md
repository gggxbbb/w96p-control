# W96P UI Redesign V5 — 设计文档

> 分支: `feat/ui-redesign-v5` | 日期: 2026-07-04

## 一、目标

将当前深色极简工业风（#1A1A18 暖调深灰、0.5px 细线、无阴影、工具感仪表盘布局）彻底重构为 **Apple Home / 戴森风格**的高端消费级体验——玻璃质感、动态氛围光晕、全身微动效、焦点式单任务界面。

## 二、视觉语言

### 2.1 配色

- **暗色主题（默认）**：纯黑底 `#000`，卡片玻璃半透明 `rgba(255,255,255,0.06)`，边框高光线 `0.5px rgba(255,255,255,0.1)`
- **亮色主题**：极浅灰白 `#F5F5F7`，卡片白底半透明 `rgba(255,255,255,0.7)`
- **强调色**：蓝紫渐变 `#5E9EFF → #A78BFA`（取代纯蓝 `#378ADD`）
- **功能色**：success/warning/danger 保持语义，饱和度降低、亮度提升

### 2.2 玻璃材质

```css
backdrop-filter: blur(20px) saturate(180%);
border: 0.5px solid rgba(255,255,255,0.1);
border-radius: 16-20px;
```

卡片带多层阴影：外发光弥散 + 内高光模拟镜面反射感。

### 2.3 字体

- 保留 MiSans（Regular/Light/Medium）
- 数值展示使用 Tabular Figures 等宽数字，避免跳变抖动
- 大字用 `font-weight: 300`（Light），标签用 Regular
- 每个档位有情绪化名称：轻柔 / 舒适 / 劲爽 / 澎湃

### 2.4 动态氛围光晕

纯 CSS 实现，`radial-gradient` + `keyframes animation`，60s 周期缓慢漂移。暗色主题紫蓝渐变光斑，亮色主题暖色调弥散。

## 三、信息架构重构

**告别 Dashboard 面板思维，改为场景驱动**：

| 旧路由 | 新设计 |
|--------|--------|
| `/` Dashboard（卡片网格） | `/` 沉浸式主控页（大圆环仪表） |
| `/fan` 风扇控制 | 合并到主控页 + 控制中心面板 |
| `/power` 电源信息 | `/power` 轻量电池/电机页 |
| `/nature-wind` 自然风曲线编辑 | `/nature-wind` 风之画卷（预制场景 + 自定义） |
| `/settings` | **删除**，打散到各处（主题切换→顶部状态区/控制中心，固件更新→整合到设备信息） |
| `/ota` | **删除**，DFU 未完成，本次不做 |
| `/history` | **删除** |
| 新增 | 控制中心（底部滑出面板，承载档位/灯光/定时/设备信息/固件升级等） |

### 3.1 底部导航胶囊

毛玻璃胶囊（约 180px 宽、48px 高），3 个图标：主控 / 电源 / 自然风。激活指示器 spring 动画滑动 + 图标微缩放。

### 3.2 控制中心（底部滑出面板）

- 占屏 70%，毛玻璃背景
- 顶部抓取条 + 三个区域：档位预设（横滚玻璃卡片）、快捷操作（行内展开）、设备信息
- 进入动画：面板 spring 弹出 + 主屏缩小到 90% 并下移

## 四、主控屏设计

### 4.1 布局（手机）

自上而下三层：

1. **顶部状态区（15%）**：设备名 + 蓝牙连接指示点（呼吸发光）+ 电量/温度旁注
2. **中央仪表区（60%）**：直径 200-240px 玻璃圆环仪表 + 64px Light 风速数字 + 档位名称
3. **底部控制区（25%）**：弧形渐变滑块 + 玻璃圆形 +/- 按钮 + 自然风胶囊开关

### 4.2 玻璃圆环仪表

- 环宽约 12px，蓝紫渐变填充表示风速
- 环内大字显示当前百分比
- 后方弥散光晕，颜色随档位变化（蓝→青→橙→紫）
- 进入时 scale spring 弹入（250ms）

### 4.3 Turbo 模式（长按交互）

- 长按圆环 0.8s 触发
- 液面变炽热橙红渐变
- 环内大字切换为跳动倒计时秒数
- 最后 5 秒加速闪烁，松手取消
- 默认 30s，可配置

## 五、大屏适配

### 5.1 平板（≥768px）

- 主环 280px 直径，居中偏上
- 两侧各悬浮一个玻璃磁贴（电池、温度）
- 控制中心占屏 50%，档位卡片 2×2 网格

### 5.2 PC（≥1024px）

- 两栏布局：左 60% 核心控制区 + 右 40% 固定控制中心面板
- 底部导航胶囊消失，改为右上角设备指示灯
- 其他页面（OTA、设置等）以浮层/模态窗口形式触发

## 六、连接体验

### 6.1 首次配对

用户点击连接 → 浏览器原生设备选择器（绕不开）→ 配对成功 → 存 deviceId → 弹设备卡片过渡动画

### 6.2 再次打开（近乎无感）

`navigator.bluetooth.getDevices()` 查缓存 deviceId → `gatt.connect()` 直连 → 设备卡片弹窗动画过渡 → 主仪表环 spring 弹入。卡片仅在视觉上包装，连接已在后台完成。

### 6.3 断连

不弹 alert/toast。仪表环光晕逐渐暗淡，数字 fade 为 `—`，底部胶囊同步变暗，连接指示点灰色。优雅的"灯灭了"氛围。

## 七、自然风页面

- 告别 128 点曲线编辑器
- 页面核心：实时生成的飘带曲线（Canvas）
- 5-6 种预制"风之场景"：山林微风 / 海边轻波 / 峡谷劲风 / 草原阵风 / 自定义
- 预制场景切换时曲线 morphing 过渡
- 自定义模式：直接在曲线上拖拽调整，类 iOS 均衡器手感

## 八、电源页面

- **电池页**：玻璃圆环电量指示 + 居中大字 + 预计剩余时间 + 底部充放电时间线
- **电机/芯片页**：玻璃行布局，背景微 fill-bar，颜色从安全绿渐变告警橙，芯片温度微型图标

## 九、动效体系

| 场景 | 动效 | 时长 |
|------|------|------|
| 页面进入 | scale spring 弹入 | 400ms |
| 页面退出 | fade + scale 0.95 | 250ms |
| 卡片 hover | translateY(-2px) + shadow expand | 200ms |
| 按钮点击 | 涟漪扩散 (ripple) | 400ms |
| 数值变化 | ease-out 缓动计数 | 800ms |
| 环液面变化 | cubic-bezier 平滑过渡 | 600ms |
| 控制中心滑出 | spring 弹性 | 500ms |
| 主题切换 | crossfade | 400ms |
| 连接弹窗 | backdrop-blur + card spring | 300ms |
| 光斑漂移 | CSS keyframe 循环 | 60s/周期 |

### 实现策略

- **动效库：`framer-motion`**（12KB gzip）——spring 物理动画、手势拖拽、layout animation、数字缓动、涟漪效果开箱即用
- **Canvas 曲线渲染：`d3-shape`**（5KB）——curveCardinal/curveBasis 自然平滑插值，飘带曲线更流畅
- **背景光斑**：纯 CSS `radial-gradient` + `animation`，不依赖 JS
- **圆环仪表**：SVG 自建（stroke-dasharray + framer-motion 驱动），替代 react-gauge-component

## 十、技术约束

- Web Bluetooth API 限制：`requestDevice()` 必须用户手势触发，无法静默扫描
- 自动重连依赖 `navigator.bluetooth.getDevices()`（Chrome 85+，用户需在 Page Info 中确认配对设备可见性）
- 现有代码几乎 100% 内联 style 无 CSS 类，重构时将提取为组件级样式体系
- 保留 Zustand store 数据层，仅重构 UI 层——协议层（`ble/`、`dfu/`）零改动
- 现有 `react-gauge-component` 不再使用，自建圆环仪表

## 十一、最终决策记录

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 动效库 | **framer-motion** | spring 物理、手势、数字缓动一体，bundle 仅 +12KB |
| Canvas 曲线 | **d3-shape** + 手写 | 5KB 换自然平滑插值，避免手动调贝塞尔 |
| OTA 页面 | **删除** | 功能未完成，本次重构不纳入 |
| 历史页面 | **删除** | 无实际价值 |
| 设置页面 | **打散** | 主题切换→控制中心/顶部区，固件信息→设备信息 |
| 路由结构 | **3 个顶层路由 + 控制中心面板** | `/` 主控、`/power` 电源、`/nature-wind` 自然风，其余通过面板/浮层承载 |

## 十二、新路由与页面结构

```
/                    → 沉浸式主控页（圆环仪表 + 弧形滑块 + 控制中心入口）
/power               → 电池/电机页（玻璃圆环 + 时间线 + fill-bar）
/nature-wind         → 风之画卷（预制场景卡片 + Canvas 飘带 + 自定义拖拽）
/battery-learn       → 保留（电池学习专用，轻量重构样式）
/debug/ble           → 保留（开发调试用，样式跟随主题）
```

**已删除路由**：`/fan`、`/power-config`、`/ota`、`/history`、`/settings`

### 控制中心面板（从主控页底部滑出）

面板内容分区：

1. **档位预设**：横滚 4 张玻璃卡片（1-4 档）+ 描述
2. **快捷操作**：自然风场景 / 灯光亮度 / 定时关机 / Turbo 时长 —— 玻璃行 + inline expand
3. **设备信息**：固件版本 / 序列号 / BLE 延迟 / 主题切换

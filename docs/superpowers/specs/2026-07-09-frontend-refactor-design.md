# W96P 控制面板前端重构设计

## 背景与问题

当前前端基于 React + Tailwind CSS 4，使用大量小卡片和 `react-grid-layout` 构建了一个信息密集的监控面板。用户反馈「太工业了」，具体体现在：

- 视觉风格硬核，像工控/调试面板。
- 信息密度过高，普通用户找不到重点。
- 操作流程不够直觉，常用功能不够突出。

## 目标

把 W96P 控制面板从「工业监控工具」改造成「消费级智能硬件 App」的体验：

- 默认首页突出最常用的风速控制。
- 支持简洁/专业双模式：普通用户一键使用，发烧友仍可查看完整数据。
- 在手机和电脑上都有良好的响应式体验。
- 不引入新框架，保持 React 19 + Vite 8 + Tailwind CSS 4 + Zustand 栈。

## 非目标

- 不改动 BLE 通信协议和数据层。
- 不添加后端服务。
- 不做完整的品牌 VI 设计，只确定基础 Design Tokens。

## 用户画像

- **普通用户**：打开 App → 调风速/档位 → 离开。不想看到电压电流。
- **发烧友/参数党**：需要实时电压、电流、温度、功率等完整数据。
- 决策：两者兼顾，默认简洁，专业数据可展开。

## 方案选择

考虑过三个方向：

1. **分层首页（推荐并选定）**：新首页 + 可展开详情面板 + 旧网格作为高级视图。
2. **渐进式仪表板**：保留卡片网格但重新设计卡片。
3. **场景优先**：首页展示场景卡片。

选择方案 1，因为它最贴合「大转盘首页」和「详情面板」的需求，同时改动可控。

## 信息架构与导航

### 页面结构

- **总览（首页）**：大转盘、档位、快捷操作、可展开详情面板。
- **自然风**：可视化曲线编辑器。
- **电源**：快充协议、C 口控制。
- **设置**：主题、PWA、关于、高级视图入口，以及电池学习、BLE 调试等不显眼文字入口。

### 导航

主导航只保留四个标签：总览、自然风、电源、设置。

- 桌面：左侧边栏，四个入口。
- 移动端：底部标签栏，四个入口，替代当前汉堡菜单 + 抽屉。

### 页面迁移

| 旧页面 | 新位置 | 说明 |
|---|---|---|
| `dashboard` | 新「总览」首页 | 旧卡片网格迁移到 `/advanced` |
| `fan` | 合并进首页 | 风速控制、档位、Turbo 已在首页 |
| `nature-wind` | 保留，视觉升级 | 曲线编辑器重做 |
| `power` | 保留，视觉升级 | 收纳原 `power-config` 内容 |
| `power-config` | 合并进「电源」 | 不再作为独立入口 |
| `battery-learn` | 设置/电源页底部文字入口 | 不 prominent，但可达 |
| `settings` | 保留 | 增加高级视图、电池学习入口 |
| `debug-ble` / `ota` / `history` | 高级视图或设置内 | 不放在主导航 |

### 高级视图与辅助入口

现有密集卡片网格不再作为首页，降级为可选的「高级/诊断视图」。入口：

- 首页详情面板底部的「打开完整面板」。
- 设置页中的「高级视图」文字入口。

电池学习和 BLE 调试不藏进高级视图，但也不放在主导航。它们以不起眼的小字/文字链接形式放在设置页底部或电源页底部，保证可达又不打扰普通用户。

## 首页设计

### 首屏布局（从上到下）

1. 顶部状态栏：设备连接状态、设备名。
2. 大风速圆环：
   - 显示当前转速百分比。
   - 支持拖拽/点击圆环调节。
   - 指针带弹性动效。
3. 档位快捷按钮：1-4 档，当前档位高亮。
4. 快捷操作区：自然风、定时、灯光、Turbo（图标 + 文字）。
5. 可展开详情面板：
   - 收起时显示「实时数据」提示。
   - 展开后显示 3-6 个关键指标（电池功率、电机转速、芯片温度等）。
   - 底部提供「打开完整面板」进入旧的高级视图。

### 交互

- 圆环：pointer/touch 事件计算角度 → 映射到 0-100 → `onChange`（实时预览）/ `onCommit`（写入设备）。
- 档位：点击直接设置对应风速。
- 快捷操作：自然风开关、定时弹窗、灯光档位、Turbo 触发。

## 视觉系统

### Design Tokens（浅色模式）

| Token | Value | Usage |
|---|---|---|
| `bg-page` | `#FFF8F0` | 页面背景 |
| `bg-surface` | `#FFFFFF` | 卡片、面板 |
| `border` | `#F0E6D8` | 卡片边框、分割线 |
| `text` | `#2C2C2A` | 主文字 |
| `text-muted` | `#888780` | 辅助文字 |
| `accent-primary` | `#FF8C42` | 主按钮、转速、档位选中 |
| `accent-nature` | `#4ECDC4` | 自然风开关、自然风页面 |
| `accent-dark` | `#292F36` | Turbo、强对比元素 |

### 字体

- 继续使用 MiSans。
- 大数字：`56px / font-weight 600`。
- 页面标题：`18px / 500`。
- 正文：`14px / 400`。
- 标签：`12px / 400`。

### 圆角与阴影

- 卡片：`16-24px`。
- 按钮/档位标签：`20px`。
- 圆环：`50%`。
- 阴影：柔和弥散，强调色按钮带同色系投影（如橙色按钮带 `rgba(255,140,66,0.25)`）。
- 边框：`1px` 浅色，不使用当前厚重的深灰边框。

### 暗色模式

暗色模式是浅色 token 的反转：

- `bg-page`: `#0F0F12`
- `bg-surface`: `#1A1A20`
- `text`: `#F4F4F5`
- `accent-primary` 保持 `#FF8C42`

### 动效

- 圆环指针：spring 弹性。
- 卡片 hover：`translateY(-2px)` + 阴影增强。
- 切换/展开：`200ms ease`。
- 档位切换：缩放脉冲。

## 组件设计

### 新增/改造组件

- `FanDial`：大风速圆环，处理自身触控几何，输出 `{ value, onChange, onCommit }`。
- `GearChips`：1-4 档快捷选择。
- `QuickActions`：自然风/定时/灯光/Turbo 图标按钮组。
- `DetailPanel`：可展开详情，内部用 `MetricRow`。
- `MetricRow`：紧凑指标（标签 + 数值 + 单位）。
- `SceneCard`：自然风页面用的场景/曲线卡片。
- `BottomNav` / `SideNav`：响应式导航。
- `ThemeProvider`：管理 light/dark/system 主题。

### 组件边界

- `FanDial` 只负责输入和显示，不直接调用 BLE；父组件通过 `onCommit` 写入。
- `QuickActions` 为纯展示组件，点击回调由父组件传入。
- `DetailPanel` 从 `useDeviceStore` 读取数据，不持有本地状态。
- 主题通过 CSS 变量和 `data-theme` 属性切换，保持与现有 `styles.css` 兼容。

## 数据流

复用现有 Zustand store：

- `useDeviceStore`：设备实时状态（转速、电池、电机、电源配置等）。
- `useConnectionStore`：连接状态。
- `useSettingsStore`：主题、首页卡片可见性等。
- `useToastStore`：操作反馈。

首页组件：

```
FanDial -> onCommit -> useBle().setFanSpeed(value)
GearChips -> useBle().setFanSpeed(calib[gear])
QuickActions -> useBle().toggleNatureWind / ...
DetailPanel -> useDeviceStore selectors
```

不新增后端，不改动 BLE SDK 包。

## 响应式策略

- **基础单位**：移动端 375-428px 优先，桌面 1024px+ 优化。
- **移动端**：
  - 底部标签栏导航。
  - 首页垂直堆叠：圆环 → 档位 → 快捷操作 → 详情面板。
  - 详情面板使用 bottom sheet 式展开。
- **桌面端**：
  - 左侧边栏。
  - 首页可左右分栏：左侧大转盘 + 档位，右侧快捷操作 + 详情面板。
  - 圆环尺寸随视口放大。
- **过渡断点**：`sm 640px`, `md 768px`, `lg 1024px`（沿用 Tailwind 默认）。

## 错误处理

- `BrowserGate`：检测 Web Bluetooth 支持，保持现有不兼容提示页。
- `ConnGuard` / `DisconnectedScreen`：未连接时显示连接引导。
- BLE 操作错误：通过 Toast 反馈，避免阻断式弹窗。
- 圆环输入边界：clamp 到 0-100。
- 暗色/浅色切换：若系统主题无法读取，回退到浅色。

## 测试策略

- 单元测试：
  - `FanDial` 角度到数值的映射函数。
  - `GearChips` 根据当前风速计算选中档位。
  - 主题切换逻辑。
- 组件测试：
  - 使用 `@testing-library/react` + `jsdom`。
  - 验证 Dial 渲染、档位点击触发回调、DetailPanel 折叠展开。
- 构建检查：
  - `pnpm build` 通过。
  - `pnpm lint` 通过。
- 手动验证：
  - 真机/模拟器上的触摸拖拽。
  - 暗色模式切换。
  - 高级视图入口可达。

## 依赖与风险

- 依赖：继续使用 `react-gauge-component` 直到高级视图重构；新 Dial 组件手写，不依赖该库。
- 风险：
  - 圆环触控在移动端的体验需要反复调试。
  - 旧高级视图与新首页的风格割裂，需要统一的视觉系统过渡。
  - 用户已养成的拖拽布局习惯被改变，需要在高级视图中保留。

## 排除范围

- 不重构 BLE 协议层和 `packages/sdk`。
- 不新增账号、云端同步、社交分享。
- 不做自然风曲线算法改动，只改编辑器 UI。
- 不重新设计 PWA 图标和启动图。
